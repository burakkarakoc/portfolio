Abstract: End-to-end (E2E) testing is a technique that tests the entire software product from beginning to end to ensure the application flow behaves as expected. The main objective is to test from the end user‚Äôs experience by simulating the real user scenario and validating the system under test and its components. Unfortunately, end-to-end tests are particularly fragile as any change in the application interface, e.g., application flow, location, or name of graphical user interface (GUI) elements, requires a change in the tests. The key to efficient, less resource-dependent test automation is to have AI/ML-aided testing systems. In this project, our ambition is to develop a learning-based approach that understands changes in the application interface and updates the test cases accordingly to ensure that the system under test is performing as designed.

I have implemented two different computer vision algorithms so that the program can find all the UI components‚Äô physical location on the screen synchronously. Generating and returning retrievable objects from the UI components‚Äô and its properties (eg. JSON).

I have written a module to conduct a test on the computer vision algorithms that we are using. The module basically compares the retrieved object from a certain CV algorithm, and actually what is in that location (eg. done by DOM parsing). In the end, returns a truth value (eg. percentage).
